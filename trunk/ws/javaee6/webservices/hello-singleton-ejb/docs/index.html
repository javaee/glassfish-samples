The EJB 3.1 Singleton Webservices Sample Application

This is a very simple EJB 3.1 Singleton webservice application

Singleton Webservice
There are 2 annotations place @Singleton and @Webservice with a single web method.


@Singleton
@Webservice
public class HelloSingletonImpl {

    public String hello();

}


@javax.ejb.Singleton is a component-defining annotation that designates this class as the bean class for a Singleton Session Bean.  
@javax.jws.Webservice is a component-defining annotation that designates this class as a Webservice Service Implementation Bean
Deployment Descriptor
The good news is the deployment descriptor is no longer required!!!  The two java files above are sufficient to completely describe this singleton webservice.  
SUN App Server Specific Deployment Configuration
There is no need to define any SUN App Server-specific deployment descrpitors (e.g. sun-ejb-jar.xml, sun-application-client.xml) for this example.  

Building, Deploying, and Running the Application

Following are the instructions for building, deploying, and running this sample application.

  1. Setup your build environment and Configure the application server with which the build system has to work by following the common build instructions.

    2. Setup some variables for Ant. The easiest way to do this is to add these variables to your-homedir/build.properties (they are populated with sample values)

    javaee.home=/glassfish
    javaee.server.name=localhost
    javaee.server.port=8080
    javaee.domaindir=/glassfish/domains/domain1

  2. hello-singleton-ejb is the sample application base directory: install_dir/ws/javaee6/webservices/hello-singleton-ejb.
  3. Change directory to hello-singleton-ejb
  4. Build, Deploy and Run the sample application using the target all

    5. hello-singleton-ejb> ant all

  5. It is possible to do the above separately by issuing separate commands as given below.

    6. app_dir> ant default compiles and packages the application

    app_dir> ant deploy deploys it to application server

    app_dir> ant run runs the test java client

  6. To run this sample, the ant targert invokes the java client code under 'test' directory. The sample can be run from either the parent directory or test directory using the targets 'run' and 'runtest' respectively.
  7. Use the target clean to undeploy the sample application and to remove the temporary directories like build and dist.

    app_dir> ant clean
