The Servlet 3.0 Annotations Sample Application

This is a simple Servlet 3.0 application using annotations for servlet, filter and content listener.

Servlet
The servlet reads the initial parameter, a request attribute and a context attribute and then output the result.

@WebServlet(name="mytest",
    urlPatterns={"/"},
    initParams={ @WebInitParam(name="message", value="my servlet") } )
public class TestServlet extends HttpServlet {
    public void init(ServletConfig config) throws ServletException {
        //read the context attribute
        ...
    }

    public void service(HttpServletRequest req, HttpServletResponse res) throws IOException, ServletException {
        ...
    }

}

@javax.servlet.annotation.WebServlet is an annotation specifying the meta-data for given servlet. In our case, it specifies the url pattern and an init-param. Note that one still need to extends javax.servlet.http.HttpServlet here.

Filter
The filter read the initial parameter and set a request attribute.

@WebFilter(urlPatterns={"/"}, initParams={ @WebInitParam(name="mesg", value="my filter") })
public class TestFilter implements Filter {
    public void init(FilterConfig filterConfig) throws ServletException {
        ...
    }

    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain) throws IOException, ServletException {
        ...
    }

    public void destroy() {
        ...
    }

}

javax.servlet.annotation.WebFilter is an annotation specifying the meta-data for given filter. In our case, it specifies the url pattern and an init-param. Note that one still need to implement javax.servlet.Filter here.

ContextListener
The context listener set a context attribute in contextInitialized() method.

@javax.servlet.annotation.WebListener
public class TestServletContextListener implements ServletContextListener {
    public void contextInitialized(ServletContextEvent sce) {
        ...
    }

    public void contextDestroyed(ServletContextEvent sce) {
        ...
    }
}

@javax.servlet.annotaton.WebListener indicates that the given class is a context listener.

There is no need to define any SUN App Server-specific deployment descrpitor (sun-web.xml) for this example.

Building, Deploying, and Running the Application

Following are the instructions for building, deploying, and running this sample application.

  1. Setup your build environment and Configure the application server with which the build system has to work by following the common build instructions.
  2. app_dir is the sample application base directory: install_dir/javaee6/web/servlet/annotation_war.
  3. Change directory to app_dir.
  4. Build, Deploy and Run the sample application using the target all

    5. app_dir> ant all

  5. It is possible to do the above separately by issuing separate commands as given below.

    6. app_dir> ant default compiles and packages the application

    app_dir> ant deploy deploys it to application server

    app_dir> ant run runs the test

  6. Note that one can run the test in one of the following way:
    • use browser to access http:<javaee.server.name>:<javaee.server.port>/annotation_war
      The following is expected in the browser:
          Hello, my servlet, my filter, my listener.
    • app_dir> ant run
    • app_dir/test> ant runtest
  7. Use the target clean to undeploy the sample application and to remove the temporary directories like build and dist.

    app_dir> ant clean
