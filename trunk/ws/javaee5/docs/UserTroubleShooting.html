Troubleshooting Sample Applications


There are three components which contribute to the smooth working of sample applications. These components are Build Infrastructure, Application Server, and the Sample Application. By following common build instructions it can be made sure that the Build Infrastruture and Application Server are setup properly. To troubleshoot the sample application, following procedure may be followed:

  1. Use the fine grained ant targets to isolate the problem. For ex. if ant all causes an exception then try to run the constituent ant targets like compile, package, deploy, and run separately.
  2. Refer to the exception in detail and try to fix it.
  3. Refer to server.log for clues/messages.
  4. Try alternative ways of doing the same thing. For ex. if deployment fails then try deploying thru GUI or CLI.
  5. Before retrying, run the clean target to undeploy and to remove the generated directories.
  6. Restart the application server which might be needed to apply certain non-dynamic configuration changes.
  7. Make sure the user id and password or the credentials are valid.
  8. If the application uses JDB resources, make sure that the database server is up and running and the resources are setup properly. Same is true for JMS resources.
  9. Use samples forum.

Please note that the above list is by no means exhaustive as it is evolving.
