Sample Applications Development Process.

Following is the process for checking out workspace, developing, testing, documenting and checking in the sample application into glassfish-samples project.

  1. Get developer access to glassfish-samples project by sending email to msreddy@dev.java.net with your glassfish user id. Once you are granted the developer access, you will be able to check out from and check in to the samples workspace.
  2. Check out the glassfish-samples project of java.net.
    cvs -d :pserver:<glassfish-userid>@cvs.dev.java.net:/cvs co glassfish-samples
  3. All the samples follow blueprints project conventions for directory structure and naming.
  4. The sample applications use blue prints build infrastructure called bp-project which is at the top level of samples directory so that it can be shared by many sample applications. The bp-project is part of blueprints java.net project on conventions.
  5. It is important that each sample application is provided with appropriate documentation and test code.
  6. Refer to the existing representative samples duke-stateful-ear and hello-stateless-ejb  to understand the directory structure, build scripts, test code, documentation.
    • duke-stateful.ear shows how to build an ear consisting of ejb and application client, and how to run application client
    • hello-stateless-ejb shows how to build an ejb module, provide test code using a java client
    • For better user experience the number of ant targets to be executed by the developer need to be kept to a minimum. Basically, the default ant target should be able to build, deploy and run the sample. All of the representative samples do this by using the ant target 'all'.
    • To understand how the entire build infrastructure works and what are various targets, pl. refer to build infrastructure documentation.
  7. For guidelines on documenting the sample, refer to the representative sample hello-stateless-ejb which is documented using the sample application documentation template.
  8. Each sample application needs one or more test cases so that the sample applications go thru some regular testing like glassfish to notice any regressions and for the purposes of promoting the samples bundle. Refer to the test directory of representative sample hello-stateless-ejb for information.

    The ant target which need to be executed for testing purpose is the 'runtest'.
  9. For setup of database and usage of jdbc resources, refer to Using Derby database with Sample applications.
  10. Once the sample application development, testing and documentation is complete, get it reviewed by a domain expert or a colleague before checking it in.
