Using Java DB with Sample applications


The main intent of this sample is to show the usage of database resources. This example shows how to start database server, stop database server, create database or delete database using sql scripts, create jdbc connection pool, delete jdbc connection pool, create jdbc resource and delete jdbc resource.

Properties for the database are defined in glassfish-samples/ws/bp-project/app-server.properties including db.home which defines where the database needs to be created. Make sure that the db.home points a directory which has write permission.

The resources needed for the project including jdbc resources are usually defined in setup directory of the project. The build.xml within setup directory will have the necessary targets for manipulating database. Usually, the resources using setup/build.xml are setup before the main application is run.

Routine database operations include:

  • Start database server
    > ant start-db
  • Stop database server
    >ant stop-db
  • Create database using create sql script
    >ant create-db
  • Delete database using delete sql script
    >ant delete-db
  • Create jdbc connection pool
    >ant create-jdbc-connection-pool
  • Delete jdbc connection pool
    >ant delete-jdbc-connection-pool
  • Create jdbc resource
    >ant create-jdbc-resource
  • Delete jdbc resource
    >ant delete-jdbc-resource
  • Create database, create jdbc connection pool and create jdbc resource
    >ant setup
  • Delete database, delete jdbc connection pool and delete jdbc resource
    >ant unsetup
