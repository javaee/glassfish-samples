The Servlet 3.1 Non-blocking-IO-write Sample Application

This is a simple Servlet 3.1 application using new API to show the Non-blocking-IO-write feature.

Description

This sample application is to show the usage of WriteListener in Servlet 3.1. There are three parts in the source, Client (org.glassfish.servlet.non_blocking_io_write_war.ClientTest), Server (org.glassfish.servlet.non_blocking_io_write_war.ServerTest), and the implementation of WriteListener(org.glassfish.servlet.non_blocking_io_write_war.WriteListenerImpl). Server sets up the WriteListener, and when data is available to be written it will send several chunks of bytes to Client. The chunk size is pre-set in order to simulate the block (the maximun writing buffer size). Client gets the data which is sent from Server and display it.

Client

In ClientTest.java, the client initiates HTTP connection to server, and waits to display the data from server.


@WebServlet(name = "ClientTest", urlPatterns = {"/"})
public class ClientTest extends HttpServlet {
    
    protected void processRequest(HttpServletRequest request,
        HttpServletResponse response) throws ServletException, IOException {
        ...
        String urlPath = "http://"
          + request.getServerName()
          + ":" + request.getLocalPort() //8080
          + request.getContextPath()
          + "/ServerTest";

        URL url = new URL(urlPath);
        HttpURLConnection conn = (HttpURLConnection) url.openConnection();
        ...
        conn.connect();
        ...
        BufferedReader input = new BufferedReader(new InputStreamReader(conn.getInputStream()));
        for (String line; input != null && (line = input.readLine()) != null;) {
            if (line.length() > 20) {
                out.println(line.substring(0, 20) + "</br>");
                out.flush();
            } else if (line.equals("---> Time out")) {
                out.println(line + "</br>");
                out.flush();
            }
        }
    }
    ...
}

Note: Since server will send several trunks of bytes, the size of byte array is larger than the capacibility of browser, so that some browsers can not display correctly. In this sample, we truncate the length of line to be 20.

Server

In ServerTest.java, once server gets the request it will start the asynchronous request process and set the WriteListener.

@WebServlet(name="ServerTest", urlPatterns={"/ServerTest"}, asyncSupported = true)
public class ServerTest extends HttpServlet {
    protected void processRequest(HttpServletRequest request,
        HttpServletResponse response) throws ServletException, IOException {
        ...
        final AsyncContext context = request.startAsync();
        final ServletOutputStream output = response.getOutputStream();
        output.setWriteListener(new ServerWriteListenerImpl(output, context));
    }
    ...
}

@javax.servlet.annotation.WebServlet(..., asyncSupported = true) is an annotation specifying the servlet name, url, and enable the async. output.setWriteListener() is to set the write listener for the input stream.
Note: Non-blocking IO only works with async request processing in Servlets and Filters.

Implementation of Write Listener

The WriteListenerImpl.java is the implementation of WriteListener.

public class WriteListenerImpl implements WriteListener {
    public WriteListenerImpl() {
        ...
    }

    @Override
    public void onWritePossible() throws IOException {
        ...
        while (output.isReady()) {
            if (System.currentTimeMillis() - startTime > 10000 || count > 10) {
                output.println("---> Time out");
                throw new Exception("Time out");
            }
            System.out.println("Writing data...... ");
            writeBytes(output, 'a');
            count++;
        }
        ...
    }

    @Override
    public void onError(Throwable t) {
        ...
        System.out.println("--> onError");
    }

    protected void writeBytes(ServletOutputStream output, char data) throws IOException {
        byte[] b = new byte[LENGTH];
        Arrays.fill(b, 0, LENGTH, (byte) data);
        output.write(b);
        ...
    }
}

onWritePossible() will be invoked when the data is available to be written to output response stream. The container will subsequently invoke the onWritePossible() if and only if isReady() returns true.
onError(Throwable t) will be invoked if ther is any error or exceptions that occurs while processing the response.
isReady() will return ture if the underlying data stream is not blocked at which point the container will invoke the onWritePossible().
Note: The constructor can be customized to handle different parameters. Usually, the parameters could be ServletInputStream, ServletOutputStream, or AsyncContext. In this sample, we are using ServletOutputStream and AsyncContext to implement the WriteListener.


Key Features

  • Involve WriteListener: Developers do not need to wait for the data by doing a loop.
  • Work with async processing: Server thread does not need to wait for the data if it is blocked or slower than the server can write

    • Building, Deploying, and Running the Application

    Following are the instructions for building, deploying, and running this sample application.

    1. Set up your build environment and configure the application server with which the build system has to work by following the common build instructions.
    2. app_dir is the sample application base directory: samples_install_dir/servlet/non-blocking-io-write-war.
    3. Change directory to app_dir.
    4. Build, deploy, the sample application using the mvn target:

      5.         You can simply use the command below to run this sample which is using Cargo framework:

              app_dir> mvn clean verify cargo:run

              You can also point Cargo at an already installed and running Glassfish:

              app_dir> mvn clean verify cargo:run -Dglassfish.home=$<glassfish_dir> (e.g. ../glassfish4)

              Note: You can also build, deploy the sample application without Cargo:

              app_dir> mvn install

              app_dir> asadmin deploy ./target/<app_name>.war

    5. You can run the sample in the following way:

      • Use the browser to access http:<javaee.server.name>:<javaee.server.port>/non-blocking-io-write-war
        The expected message in the browser is as follows:
            This is the data from Server.
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          aaaaaaaaaaaaaaaaaaaa
          ---> Time out
          Check server log for more details
    6. Use the glassfish command line to undeploy the application.

      app_dir> asadmin undeploy <app_name>

    7. Use the target clean to remove the temporary directories like /target.

      app_dir> mvn clean

    Building, Deploying, and Running the Application in NetBeans IDE

    Perform the following steps to build, deploy, and run the application using NetBeans IDE:

    1. Refer to the common build instructions for setting up NetBeans IDE and Java EE 7 SDK.
    2. In the NetBeans IDE, choose File → Open Project (Ctrl-Shift-O), navigate to the samples_install_dir/servlet/ directory, select non-blocking-io-write-war, and click Open Project.
    3. In the Projects tab, right click non-blocking-io-write-war and select Run to build, deploy, and run the project.
