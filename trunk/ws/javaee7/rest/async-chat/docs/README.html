JAX-RS Asynchronous Chat Sample

Sample using JAX-RS asynchronous features to implement simple producer/consumer chat.

Description

This sample shows usage of javax.ws.rs.container.AsyncResponse. Deployed resource ChatResource contains two methods which both works in asynchronous mode. They contain @AsyncResponse annotated asyncResponse that causes to run these methods in the suspended mode. In this mode the response is not returned from the resource method as a return value but must be resumed by calling AsyncResponse.resume() method. Before the response is resumed the execution thread is returned back to container.

Resource method receiving the message (ChatResource.getMesssage()) stores the async response into a blocking queue. After the message is send to server (ChatResource.posMesssage()) the async response of request waiting for message is taken from the queue and it is resumed with the message. Instead of keeping messages in the queue the responses waiting for these messages are kept and messages are directly delivered to them when they are available.

Front page shows the text input field for message. Enter the message and click POST MESSAGE. The background asynchronous POST request will be invoked to server where it will be blocked on the blocking queue until any message is requested. Then click GET MESSAGE which also invokes background asynchronous GET request to the server. This will be suspended and afterwards resumed with the message from the POST request that is still waiting blocked on the blocking queue. Message will be returned to the browser and message is displayed in the box below the GET MESSAGE button. When buttons are clicked in the opposite order the GET request will be suspended waiting to be resumed by incoming POST request.

The screen also contains the field with the status of an asynchronous queue kept in the ChatResource. This field is automatically refreshed in short intervals by calling the GET on /chat/queue from ChatResource.

Only one GET and one POST request can be sent to the server (then buttons are disabled). In order to submit more GET and POST requests more browser windows needs to be opened. The screen also contains log of asynchronous requests done by browser.

Key Features

  • @AsyncResponse
  • @Path, @Singleton

Building, Deploying, and Running the Application

Perform the following steps to build, deploy, and run the application:

  1. Set up your build environment and configure the application server with which the build system has to work by following the common build instructions.
  2. samples_install_dir is the sample application base directory. Go to: samples_install_dir/javaee7/rest/async-chat.
  3. Build, deploy, and run the sample application using the run outcome.

    mvn clean verify cargo:run

  4. Front page of this sample is at http://localhost:8080/async-chat (port number might vary)
  5. Use the clean outcome to undeploy the sample application and to remove the temporary directories such as build and dist.

    mvn clean
