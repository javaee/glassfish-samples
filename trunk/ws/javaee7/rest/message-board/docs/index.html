The JAXR-RS MessageBoard Application

This sample demonstrates the possibility to use no-interface EJB session beans as RESTful resource classes deployed as a servlet.

MessageBoard represents message boards you may know from aiports or train stations. It stores messages and every new added message is placed on top and gets an unique message id.

If configured properly, page http://localhost:8080/message-board will be opened after invoking the run target.

The mapping of the URI path space is presented in the following table:

URI path Resource class HTTP method Description
/message-board --- GET Simple html/ajax client
/message-board/app/messages MessageBoardResourceBean GET Returns first ten messsages
/message-board/app/messages/{id} MessageBoardResourceBean GET Returns message with uniqueId = {id} or returns HTTP 404 if message does not exist
/message-board/app/messages/{id} MessageBoardResourceBean DELETE Deletes message with uniqueId = {id} or returns HTTP 404 if message does not exist
/message-board/app/messages/
(with message as request entity) 		MessageBoardResourceBean POST Creates new message and allocates unique id for it

Application Structure

The application is built and deployed as a WAR file.

Web Module

The web module consists of:

  • An HTML page that represents the view layer of the application
  • A simple class that represents a message
  • Two providers - MessageBodyWriter implementations for Message and ListMessage, which are also a stateless EJB session or singleton bean.
  • Two no-interface-view session beans that are simulatenously REST resources
  • A singleton bean that holds data "persistently" in application instance
  • Response filter that add the header previous-response and this-response
  • Reader interceptor that intercepts incoming entity and removes all invalid characters. Valid characters are characters, numbers, white spaces and dot.
The starting point of the application is the main index.html page (javascript must be enabled). The application control is straightforward:
  1. Click the LIST ALL MESSAGES button to see all messages. This command is executed automaticaly when you hit the main page.
  2. Select message number and click the GET MESSAGE button to see one particular message.
  3. Select message number and click the DELETE MESSAGE button to delete one particular message.
  4. Fill in message input box and click the ADD MESSAGE button to add message. Message will be added as first. Use any not valid characters (e.g. '/*+') to check that they will be removed by interceptors.
The page contains section response headers that displays headers of received response. Filters should add correctly last-response and this-response.

Key Features

  • jax-rs: Filter, Interceptor, ExceptionMapper, MessageBodyWriter
  • jax-rs annotations: @Path, @Singleton, @GET, @POST, @DELETE
  • ejb: @EJB, @Stateless, @Singleton

Building, Deploying, and Running the Application

Perform the following steps to build, deploy, and run the application:

  1. Set up your build environment and configure the application server with which the build system has to work by following the common build instructions.
  2. samples_install_dir is the sample application base directory. Go to: samples_install_dir/javaee7/rest/message-board.
  3. Build, deploy, and run the sample application using the run outcome.

    mvn clean verify cargo:run

  4. Front page of this sample is at http://localhost:8080/message-board (port number might vary)
  5. Use the clean outcome to undeploy the sample application and to remove the temporary directories such as build and dist.

    mvn clean
