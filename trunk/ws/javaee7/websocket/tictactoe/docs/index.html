The Tictactoe WebSocket Sample Application

This is a sample application using the Java API for WebSocket.

Description

This is a simple tictactoe game application which demonstrates how to use the basic WebSocket APIs for both client side and server side in JSR356. This sample is also using JavaFx platform to build the client endpoint (org.glassfish.samples.tictactoe.client). Server side (org.glassfish.samples.tictactoe.server) is only using the normal WebSocket APIs. Two clients create WebSocket connections to server which looks like a hub, dealing with the interactions between two clients. Once there is a winner, the connection session will be closed by client side, and onClose() from server side will be invoked. The main purpose of this sample is going to show how to use WebSocket Client API, and how to create connection between standalone front end applications and server.

Note: In this sample, we are not going to use HTML Javascript API to create the WebSocket connection. However, we are using WebSocket Client APIs instead.

Key Features

This sample demonstrates the following key features:

  • In org.glassfish.samples.tictactoe.client.TicTacToeClient.java

    ...

    Instead of using Javascript APIs, this sample is using WebSocket Client API to create WebSocket connection

  • In org.glassfish.samples.tictactoe.client.LocalEndpoint.java

    ...

    @Override
    public void onOpen(Session session, EndpointConfig config) {
    ...
    }

    @Override
    public void onMessage(final String message) {
    ...
    }

    ...

    Two metods are overridden in client endpoint which have the similar funcionalities with Javascript onOpen and onMessage APIs. onOpen will be invoked once the WebSocket connection is established. onMessage will be invoked once there is any messages being sent to client side.

  • Server side is using WebSocket annotations which is demostrated in other samples. And basic WebSocket APIs, onOpen, onClose and onMessage. CloseReason is used to display the reasons why the connection is closed.

Building, Deploying, and Running the Application

Perform the following steps to build, deploy, and run the application:

  1. Set up your build environment and configure the application server with which the build system has to work by following the common build instructions.
  2. app_dir is the sample application base directory: samples_install_dir/javaee7/websocket/tictactoe.
  3. Build, deploy, and run the server application using the run outcome.

    cd server

    mvn clean verify cargo:run

  4. Running the client standalone application. Note: We need two client endpoints, so you may need to do the operations below twice in two different terminal.

    cd app_dir/client

    mvn clean install

    Note: If the sample is running successfully, you will get two dialogs which are two 3*3 chessboards. And just follow the UI instruction to play the tictactoe game.
             If you get any exceptions from JavaFX, you may need to check if the client pom.xml is using the correct PATH to jfxrt.jar

  5. Use the clean outcome to undeploy the sample application and to remove the temporary directories such as build and dist.

    mvn clean

Note: Running the sample in Netbean, you can just open the sample application( Both client and server) and use the UI to build and run.
